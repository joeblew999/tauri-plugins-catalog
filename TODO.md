# TODO: Tauri toolchain — nushell scripts

> Goal: a small set of nushell scripts that *install*, *verify*, and
> *troubleshoot* the Tauri 2 toolchain idempotently. Two layers:
>
> 1. **Base** — every Tauri project, every dev VM.
> 2. **Android** — adds the mobile-target stack on top of the base.
>
> **Build model:** target-specific dev environments. No host-to-host
> cross-compile of *desktop* targets (no `xbuild`, no `cross`) — Linux
> binaries are built on Linux, macOS on macOS, Windows on Windows. Android,
> which is never the host OS, is built from any desktop host via NDK
> (macOS-arm64, macOS-x86_64, or Linux-x86_64 — all fine). iOS still
> requires a macOS host (Apple's rules).

**Status:** scoping. No scripts written.

Repo location is `scripts/tauri/` with mise tasks under the `tauri:` namespace.
Whether this eventually splits into its own repo (`tauri-toolkit`?) is an open
question — defer until `doctor.nu` exists.

---

## Layers

### Layer 1 — Base (every Tauri project, every VM)

| piece | install path | notes |
| ----- | ------------ | ----- |
| Rust ≥ 1.77.2 | `mise use rust@stable` | Tauri 2 MSRV |
| Node LTS | `mise use node@lts` | for JS frontends, `@tauri-apps/api` |
| tauri-cli | `mise use cargo:tauri-cli@^2` | declarative under mise (vs `npm:@tauri-apps/cli`) |
| cargo-binstall | `mise use cargo:cargo-binstall` | accelerator for any other `cargo install` |
| OS webview deps | OS package manager | webkit2gtk + libsoup + librsvg on Linux; Xcode CLT on macOS |

### Layer 2 — Android (any desktop host)

| piece | install path | notes |
| ----- | ------------ | ----- |
| JDK 17 | `mise use java@17` | system Java often wrong version; mise pins it |
| cmdline-tools | `mise use asdf:mise-plugins/mise-android-sdk@latest` | gets `sdkmanager` on PATH |
| platforms, build-tools, NDK | `sdkmanager` (from layer above) | versions pinned to whatever Tauri 2 requires |
| Rust Android targets | `rustup target add aarch64-linux-android armv7-linux-androideabi i686-linux-android x86_64-linux-android` | 4 triples |
| `.cargo/config.toml` linker stanzas | generated by `init-project.nu` | per-target linker pointing at NDK toolchain |
| Dev keystore | generated by `init-project.nu` | for build signing |
| adb + emulator | included with SDK | for local verification |

Layer 2 runs on any desktop host (macOS or Linux). Google publishes
cmdline-tools and NDK for both; the mise plugin picks the right one.

iOS layer is out of scope for v1; would need a macOS host and a separate
`init-project-ios.nu`.

## Why the PITA, layer by layer

### Layer 1
- **tauri-cli compile time**: first `cargo install tauri-cli` takes 5–10 min.
  Whether mise's `cargo:` backend uses cargo-binstall transparently is unclear
  (research). If not, we install cargo-binstall first and use it directly.
- **OS webview deps**: package names differ per distro; `apt install
  libwebkit2gtk-4.1-dev libsoup-3.0-dev` (Ubuntu 24.04) vs older names on
  22.04. Missing them gives a cryptic build error. `doctor.nu` should detect.
- **MSRV drift**: Tauri's minimum Rust version creeps up. Pin via mise.

### Layer 2
- **JDK conflicts**: system Java is often Java 8 or 21; gradle wants 17.
  `JAVA_HOME` must be visible to all subprocesses (gradle spawned by Tauri).
- **cmdline-tools bootstrap dance**: download zip → unpack into the specific
  nested layout sdkmanager expects → accept licenses. Easy to get wrong.
- **NDK version pinning**: Tauri 2 expects a specific NDK major. We need to
  find where it's pinned and match.
- **Linker stanzas**: `.cargo/config.toml` needs per-target linker pointing at
  the NDK toolchain. Path varies by NDK version.
- **Env-var leakage**: gradle subprocess started by Tauri needs `JAVA_HOME` /
  `ANDROID_HOME` / `NDK_HOME`. mise activates shell, but Tauri spawning gradle
  is the common breakage point.
- **Signing keys**: keystore generation + integration with `tauri.conf.json`.
- **Emulator**: Android Studio's emulator works on any desktop host; Linux
  in a VM needs nested KVM (may or may not work). Physical device via adb is
  always an option.

## Sketched script set

`scripts/tauri/` — flat layout, prefixed names to imply layer. mise tasks
under `tauri:` namespace.

| script | mise task | purpose | runs when |
| ------ | --------- | ------- | --------- |
| `doctor.nu` | `tauri:doctor` | Verify every prerequisite that applies. Auto-detects whether project targets Android. Prints red/green per item with remediation command. Idempotent, no side effects. | any time something is broken |
| `bootstrap-base.nu` | `tauri:bootstrap` | Install rust + node + tauri-cli + cargo-binstall + OS webview deps. Idempotent. | one-shot per VM |
| `bootstrap-android.nu` | `tauri:bootstrap:android` | On top of base: JDK 17, cmdline-tools, sdkmanager-driven NDK/platforms/build-tools, Rust Android targets. Works on any desktop host. | one-shot per host |
| `init-project.nu` | `tauri:init` | Inside a Tauri project: ensure `tauri android init` has been run, generate `.cargo/config.toml` linker stanzas, dev keystore. | per project |
| `build.nu` | `tauri:build` | Wrap `cargo tauri build` / `android build` with sanity checks that catch known failure modes early. | per build |

`doctor.nu` is the entry point a frustrated dev runs first. Every other script
should leave the system in a state where `doctor.nu` says green.

## Out of scope (v1)

- **Host-to-other-desktop-host cross-compile.** Linux binaries built on
  Linux, macOS on macOS, etc. Use VMs for target OS coverage. (Android via
  NDK from any desktop host is in scope; that's not what this rule excludes.)
- **Windows host.** Revisit later.
- **iOS.** Different toolchain, needs macOS host. Separate effort once
  Android layer is stable.
- **CI builds.** Same toolchain inputs, different shape (matrix builds, cache).
  Separate effort.
- **Replacing `tauri android init` itself.** Wrap it.

## Answered

- mise's `cargo:tauri-cli` is the slow source-build path. mise registry has
  `cargo-binstall` aliased to *both* aqua (prebuilt, fast) and cargo
  backends; default resolves to aqua. So `bootstrap-base.nu` should install
  `cargo-binstall@latest` first via aqua, then `cargo binstall tauri-cli` to
  skip the source build of tauri-cli.
